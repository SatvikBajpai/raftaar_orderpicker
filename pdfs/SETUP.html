Quick Installation

Method 1: Download and Setup (Recommended)

  1. Download the Application

    # Option A: Clone from repository
git clone <repository-url>
cd raftaar_orderpicker

# Option B: Download ZIP file
# Extract to your desired location

  2. Get Google Maps API Key

    • Go to Google Cloud Console
    • Create a new project or select existing one
    • Enable the following APIs:
      • Maps JavaScript API
      • Directions API
      • Places API
      • Geocoding API
      • Routes API (recommended)
    • Create an API key
    • Add API key restrictions for security

  3. Configure API Key

    • Open index.html
    • Find line 9 (Google Maps script tag)
    • Replace AIzaSyDr-DYch4S-JeL-sTlbixPKSjgOJiHLe8A with your API key:
    <script async defer src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY_HERE&libraries=routes,geometry,places&callback=initMap"></script>

  4. Start Web Server

    Option A: Python (if installed)

    # Python 3
python3 -m http.server 8000

# Python 2 (legacy)
python -m SimpleHTTPServer 8000

    Option B: Node.js (if installed)

    # Install http-server globally
npm install -g http-server

# Start server
http-server -p 8000

    Option C: PHP (if installed)

    php -S localhost:8000

  5. Access Application

    • Open browser and go to http://localhost:8000
    • Application will load and initialize automatically

Detailed Configuration

Google Maps API Setup

  1. Create Google Cloud Project

    1. Visit https://console.cloud.google.com/
2. Click "Select a project" ‚Üí "New Project"
3. Enter project name: "Raftaar Order Picker"
4. Click "Create"

  2. Enable Required APIs

    1. Go to "APIs & Services" ‚Üí "Library"
2. Search and enable each API:
   - Maps JavaScript API
   - Directions API
   - Places API
   - Geocoding API
   - Routes API (for advanced routing)

  3. Create and Secure API Key

    1. Go to "APIs & Services" ‚Üí "Credentials"
2. Click "Create Credentials" ‚Üí "API Key"
3. Copy the generated key
4. Click "Restrict Key" for security:
   - Application restrictions: HTTP referrers
   - Add your website domains
   - API restrictions: Select only enabled APIs

Store Location Configuration

Update your store coordinates in js/main.js (line 6-9):

this.storeLocation = {
    lat: 21.142639,  // Replace with your store latitude
    lng: 79.090385   // Replace with your store longitude
};

To find your coordinates: 1. Open Google Maps 2. Right-click on your store location 3. Click on the coordinates to copy them

Security Configuration

API Key Security

  1. Restrict by Domain

    In Google Cloud Console:
1. Go to Credentials ‚Üí Edit API Key
2. Application restrictions ‚Üí HTTP referrers
3. Add: yourdomain.com/* and www.yourdomain.com/*

  2. Restrict by API

    API restrictions:
Maps JavaScript API
Directions API
Places API
Geocoding API
Routes API
(Uncheck all others)

HTTPS Requirements

  • Local Development: HTTP is fine
  • Production: HTTPS required for geolocation features
  • Google Maps: Works on both HTTP (localhost) and HTTPS

Troubleshooting Installation

Common Issues

Issue: Map not loading

Solution:
1. Check API key is valid
2. Verify APIs are enabled in Google Cloud
3. Check browser console for errors
4. Ensure correct domain restrictions

Issue: ‚ÄúThis page can‚Äôt load Google Maps correctly‚Äù

Solution:
1. Check API key restrictions
2. Verify billing is enabled (Google requires billing account)
3. Check API quotas aren't exceeded

Issue: CORS errors

Solution:
1. Don't open index.html directly (file://)
2. Always use a web server (http://)
3. For Google Sheets: ensure sheet is public

Issue: Customer data not loading

Solution:
1. Verify Google Sheets is publicly accessible
2. Check spreadsheet ID in customerManager.js
3. Ensure correct column format
4. Check browser console for errors

Performance Issues

Slow map loading

Solutions:
1. Reduce initial zoom level
2. Implement lazy loading for markers
3. Use marker clustering for many orders
4. Optimize API calls

High API usage

Solutions:
1. Implement request caching
2. Use batch geocoding
3. Monitor API quotas
4. Implement rate limiting
